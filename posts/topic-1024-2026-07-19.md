# 3步用 Clawk 给 Cursor 搭隔离沙箱：AI 随便删文件也不怕

最近 Cursor 被爆出有个安全漏洞，恶意代码能通过 AI 编程直接在你电脑上执行。这事儿在我们几个搞独立开发的朋友群里炸锅了。说实话，搞 AI 编程，谁没经历过 Agent 一抽风把本地文件改飞的惨剧？今天咱就聊聊怎么用 Clawk 给 AI 隔离出一个 Linux 沙箱环境，彻底解决这个隐患。

下面这段对话，是昨天我和后端老李在群里聊天的复盘。老李最近用 Cursor 越来越猛，但也越来越慌。

老李：兄弟们，现在用 Cursor 和 Claude Code 真是又爱又怕。刚才它自作主张帮我清理依赖，直接把我的 `.env` 配置文件给抹了。我找半天都不知道为啥数据库连不上。

我：你这算轻的。我前几天让 Agent 帮我重构一个 Node 项目，它为了清理所谓的无用文件，执行了个 `rm -rf` 把整个未推送的本地分支给清空了。一天白干。AI 编程安全这块，真不能由着它性子来。

老李：那咋弄？开虚拟机？太重了，起一个要等半天，还得分配内存。用 Docker 跑 Cursor？配置网络和卷映射能折腾一下午，我不嫌麻烦 AI 还嫌慢呢。

我：所以我最近换了个思路，用 Clawk。这玩意就是个给 AI Agent 专门做隔离的轻量级沙箱。它直接给你分配一个一次性的 Linux 虚拟机环境，跑完即弃。

老李：一次性的？听起来挺费资源啊。而且它怎么知道我什么时候写完代码了？

我：其实还好，它是按需启动的。你在里面装环境、跑脚本、删文件，AI 怎么折腾都行，反正弄完就丢。你的宿主机文件系统它碰都不碰。这就相当于给 AI 穿了件防弹衣，它自己死活不管，别把你的主程序带崩就行。

老李：那我的代码怎么同步进去？它在一个虚拟机里，我在外面写？

我：Clawk 启动的时候会自动把当前目录挂载进去。你在 Cursor 里写代码，Agent 觉得要跑个 `npm run dev` 试试水，它其实是在那个临时的 Linux 环境里跑的。不会污染你本机全局的 Node 版本和包。

老李：听着有点意思。赶紧说说怎么装，我现在手头有个 Next.js 的路由 bug 想让 AI 修，但真不敢让它瞎跑命令了。

我：如果你是 Mac，直接用 Homebrew，终端里敲 `brew install clawk` 就行。如果你习惯用 npm 全局装，就 `npm install -g @clawk/cli`。

老李：装好了，然后呢？Cursor 怎么连过去？

我：你先在终端里把 Clawk 跑起来，它会给你一个隔离的 shell 入口。然后打开 Cursor，点左上角苹果图标旁边的菜单 -> Settings -> Tools -> Terminal。在那个 Shell path 或者 integrated default profile 里面，把 Clawk 提供的入口脚本路径填进去。这样 Cursor 里面自带的终端，就直接通向沙箱了。

老李：我去试试。打开了 Next.js 项目，按 Cmd+I 唤出了 Composer。

我：还是跟它说「帮我修一下这个接口报错」？

老李：对。它现在开始读文件了。

我：你盯着点它的动作。如果是平常，AI 找不到原因，可能会直接尝试改你的 `package.json`，然后跑 `npm install` 装新包。万一它装的版本跟你本地其他项目冲突了，你本机环境就崩了，得花半天时间弄 npm 的依赖树。

老李：哎对，它现在说要装个 `next-auth` 试试。它在沙箱里跑 `npm install` 了。我本机没动静！

我：这就对了。它在沙箱里装包测试，觉得不对劲它会自己回滚或者换版本，折腾的全是那个一次性的 Linux。它要是修不好，直接把沙箱关了，干干净净。

老李：那它测完没问题了，我怎么把改动拿回我本地真实环境？

我：不用你操心。Clawk 会把代码的改动实时双向同步到你本地目录的。你照旧在 Cursor 右侧的代码审查界面看 diff，觉得没问题点 Accept 就行。代码是落地的，但那些它为了测试装的乱七八糟的包，全留在了沙箱里。

老李：那如果我让它跑测试脚本，它在沙箱里生成了个 coverage 测试报告文件，我本机能看到吗？

我：看得到。只要是工作区目录里的文件变动，都会同步过来。它删库跑路删的也是沙箱里的运行库，你的原始数据毛都不掉一根。

老李：这思路绝了。尤其现在 Claude Code 这种直接接管终端的 Agent，它自己决定执行什么命令，风险极高。你刚才说 Cursor 有个什么安全漏洞来着？

我：前阵子爆出 Cursor 有个 0day 安全事件，攻击者能构造特定的提示词或者代码，让 AI 去读取你电脑里的敏感文件，比如 SSH 密钥，甚至直接往外发。你想，AI 直接在你本机跑命令，权限多大啊。有了沙箱隔离，它顶多读个沙箱里的假密钥，根本碰不到你宿主机的 `~/.ssh`。

老李：懂了。我这就把所有项目的终端都换成 Clawk。万一 AI 在里面把环境搞坏了怎么办？

我：坏了就丢掉呗。Clawk 重启一下又是一个干净的 Linux。你再也不用为了清理 Agent 装的乱七八糟的全局依赖发愁了。这个坑我踩过太多次了。

老李：行，我去把那个 Next.js 项目跑起来试试水。

我：去吧。Agent 跑完，觉得没问题了，记得在 Cursor 里检查一遍 diff 再 commit。沙箱防的是破坏环境，不防 AI 写出烂代码。
