还在复制粘贴写重复代码？用 Claude Code 自动化编程只需 3 分钟

每次建新项目都要手写一堆配置文件？重构老代码时改了东边忘西边？这些枯燥又容易出错的重复劳动，其实完全可以交给 AI 来干。今天教你用 Claude Code 这个命令行神器，把重复编程任务彻底自动化。

**Claude Code** 是 Anthropic 官方推出的终端 AI 工具。它不是网页聊天框，而是直接运行在你的终端里，能读写本地文件、执行命令的 AI 编程助手。

接下来，我们通过三个真实场景，看看它怎么帮你把几小时的工作压缩到几分钟。

**场景一：批量生成样板代码**

做后台系统时，给 5 个数据表写 CRUD 接口是最烦人的体力活。现在，你只需要告诉 Claude Code 你的需求。

① 打开终端，进入项目根目录，输入 `claude` 启动工具。
② 直接输入你的需求指令。

```text
请根据 models 文件夹下的 User、Order、Product 模型，生成对应的 CRUD 控制器和路由文件。
要求：
1. 遵循项目现有的 Express 代码风格
2. 包含基础的参数校验
3. 将新文件保存到 controllers 和 routes 目录下
```

③ Claude Code 会自动分析现有代码风格，创建文件并写入代码。

[截图：终端中 Claude Code 自动生成文件并询问是否确认的界面]

**场景二：跨文件智能重构**

改一个公共函数的传参，要在几十个文件里找调用方逐一修改，很容易漏掉。

① 告诉 Claude Code 你的重构目标。

```text
找出 src 目录下所有调用了 getUserInfo(userId) 的地方，将参数改为 getUserInfo(userId, fields)。
要求：
1. 在调用处补充 fields 参数为 ['name', 'email']
2. 先列出需要修改的文件清单，等我确认后再执行修改
```

② 审核它列出的文件清单，输入 `y` 确认，它会一次性完成所有修改。

[截图：Claude Code 列出修改清单等待用户确认的界面]

**场景三：自动补全单元测试**

写完业务代码不想写测试？让 AI 替你兜底。

① 直接下达测试指令。

```text
分析 utils/calculate.js 中的所有导出函数，为每个函数生成 Jest 单元测试。
要求：
1. 覆盖正常输入、边界情况和异常输入
2. 将测试文件保存到 __tests__ 目录
3. 生成后自动运行 npm test，如果报错请自行修复测试代码
```

② 它会写好测试，跑一遍，甚至能自己修好跑不通的用例。

[截图：Claude Code 自动运行测试并通过的终端输出]

**效果对比**

传统方式下，批量写 5 个模块的 CRUD 接口加测试，至少需要 2 小时，还容易漏掉参数校验。用 Claude Code，从下达指令到生成并通过测试，只要 3 分钟，代码风格还高度统一。

**小贴士**

1. 让 AI 先列计划：在 Prompt 里加上“先列出修改计划，等我确认后再修改文件”，能有效防止它乱改代码。
2. 敏感信息避坑：不要在包含生产环境密钥的项目目录里运行，它有本地文件读写权限，避免核心配置泄露。
