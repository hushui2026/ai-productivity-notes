# 用 Kimi 写竞品分析踩了 3 个坑，靠「自我纠错 Prompt」让 AI 自己改错

上周三快下班，老板甩给我一份行业研报，让我第二天早上交一份针对某个竞品的深度分析。我心想这还用说，果断打开 Kimi 准备搞定。十分钟不到，洋洋洒洒两千字出来了，排版工整，小标题起得比我头发还密，看着特别像那么回事。我连看都没仔细看，直接 Ctrl+C 然后Ctrl+V 交了差。结果第二天开会，老板当着全组的面，指着报告里的一段数据问我：“这个增长率你从哪看的？人家财报里根本没提！”我当时后背直接湿了，结结巴巴解释说是查的资料。

说实话，用 AI 生成内容最大的坑就在这——它一本正经地胡说八道，你根本看不出来。后来我痛定思痛，翻遍了各种高阶玩法，才搞明白一个叫「自我纠错 Prompt」的技巧。简单说，就是别指望 AI 一次就对，你得用 Prompt 逼着它自己查错、自己改。这中间我又踩了俩坑，今天全盘托出，你们别再走弯路了。

第一个坑，盲目相信 AI 的“自信”。那天下班后我重新跑了一遍那个竞品分析。AI 给出的文字极其笃定，用了诸如“显而易见”、“据统计”这种词。我当时的问题出在 Prompt 太简单，就写了一句“帮我写一份XX竞品分析”。AI 为了迎合我，它宁可编数据也要把话说圆满。这其实是它底层的逻辑在作祟，概率模型嘛，怎么顺口怎么来，它并不理解什么是“事实”。我后来发现，必须在 Prompt 里加上一条限制，让它知道自己“可以犯错”，但必须标明出处。如果你不给这个压力，它就会顺滑地滑向胡编乱造。比如我那次，它硬生生给竞品编造了一个“用户留存率高达 85%”的数据，看着特别唬人。你如果不具备相关背景知识，百分之百会被它带沟里去。

发现数据造假后，我当然很生气，就在对话框里回它：“你写的不对，数据是假的，重新写。”这就迎来了我的第二个坑——只会骂它“不对”，但不会引导它自查。结果呢？AI 道歉倒是挺快，说“抱歉，我更正一下”，然后把我原本的文案打乱顺序又发了一遍，核心的假数据还在，只是换了个说法。这就好比你让实习生改方案，只跟他说“感觉不对，重写”，他除了瞎猜你的心思还能干嘛？AI 也一样。你不告诉它标准是什么，它连自己错在哪都不知道。这个坑浪费了我一晚上时间，我一直在跟它玩“你改我挑”的打地鼠游戏。它每次改一点，改完又冒出新的问题，我在这头不断地挑刺，最后改得面目全非，连原本写对的部分都被带歪了。

到周四半夜，我彻底急了。这引出了第三个坑：一口气让它跑到底，没有留纠错的空间。我把所有要求揉在一个超长 Prompt 里，让它一次性生成完整报告，甚至连格式都规定死了。AI 很给力，直接吐了三千字。但我一看，开头说竞品优势在价格，结尾又说要主打高端，前后矛盾。这时候再去改，上下文已经全乱套了。你发一句“前后矛盾”，它改了结尾，接着开头又崩了，牵一发而动全身。我才琢磨过味来，用 AI 干活，绝对不能搞“一锤子买卖”。你给它太大的自由度，它就会在逻辑上放飞自我。长文本生成是 AI 的弱项，越往后写，注意力越分散，出现幻觉和逻辑断裂的概率就越大。

摔了这三个跟头后，我总算是摸清楚了。让 AI 生成内容不难，难的是怎么让它“长个脑子”审视自己。这就是自我纠错 Prompt 派上用场的地方。与其我们人肉去挑错，不如把挑错的任务也丢给 AI。核心思路分三步：先让它在输出前自查，再给它具体的审查标准，最后逼它输出一份错误报告。别觉得这听起来很高级，其实就是在 Prompt 里加几句话的事。讲真，我第一次这么搞的时候，都被效果惊到了。

我当时改写了 Prompt，把直接要结果的指令，改成了带审查机制的。大概长这样，你可以直接复制去试：

```markdown
你是一个资深的行业分析师。请帮我撰写一份关于 [你的竞品名称] 的竞品分析报告。

在正式输出报告之前，请按照以下步骤进行自我纠错：
1. 草稿生成：先在内心生成一份草稿，不要输出。
2. 事实核查：检查草稿中的所有数据、事实声明是否有依据。如果有不确定的信息，请直接标注[待核实]，不要自行编造。
3. 逻辑自查：检查报告前后是否存在矛盾，例如优缺点是否冲突，结论是否由论据推导得出。
4. 输出报告与错误日志：输出最终的报告，并在报告末尾附上“### 自我纠错日志”，列出你在步骤2和3中发现的潜在问题及修改理由。
```

把这段 Prompt 喂进去之后，AI 的反应截然不同了。它先吐出了报告，然后在最下面老老实实地附了一段纠错日志。日志里写着：“原本草稿中提到该竞品市场份额达 30%，但缺乏确切来源，现已修改为定性描述‘市场份额较高’；原本结论部分与优势分析存在轻微矛盾，已调整结论的侧重点。”那一刻我真想拍大腿，这不就是我想要的效果吗？它自己把雷给排了。不用我再逐字去抠，肉眼可见地顺手了。

后来我又把这个技巧用到了写周报上。平时写周报，最容易犯的毛病就是流水账，干了什么写一堆，但没体现出价值。我就用自我纠错的思路，写了个专门对付周报的 Prompt：

```markdown
你是我的直属领导，请帮我润色以下周报内容：[你的周报草稿]

请进行自我纠错：
1. 价值提炼：检查是否全是流水账，没有结果导向。如果有，请将“做了什么”改为“达成了什么效果”。
2. 数据量化：检查是否缺少量化指标。对于模糊的描述（如“很多”、“有效”），请提示我补充具体数据。
3. 风险预警：检查是否有未闭环的工作，如果有，请在周报末尾补充“下周跟进计划”。
4. 输出润色后的周报，并附上“### 领导视角点评”，指出这周报哪里看着最没说服力，哪里需要补充细节。
```

你猜怎么着？AI 拿着我那堆流水账草稿，硬是给我挑出了三个“缺乏数据支撑”的毛病，还用领导的口吻点评说：“这个项目推进中遇到的阻力描述过于主观，建议补充客户的具体反馈。”看完我直冒冷汗，要是原封不动发出去，肯定又要被骂。改完之后交上去，老板没挑出毛病，还夸我最近汇报逻辑清晰了不少。

当然，这里还有个要注意的地方，就是你给的自查标准得具体。你要是只写“请检查是否有错”，那就等于没写。你得把你的行业经验塞进去。比如你做财务的，就加上“请检查资产负债表是否配平”；你做程序的，就加上“请检查代码是否有死循环”。把你的专业 Checklist 变成 Prompt 里的一行字，AI 就成了你最得力的干将。别问我是怎么知道的，都是熬夜换来的教训。而且，千万别图省事直接让它出结果，那个“先在内心生成草稿”的指令特别重要。这就相当于给 AI 留了思考的缓冲带，让它先打个腹稿，再自己审一遍，而不是张口就来。这个机制在学界好像叫思维链什么的，咱不管那些名词，只要知道好用就行。

用 AI 真的不是把问题甩给它就完事了。你得当个教练，教它怎么审视自己的工作。好的 Prompt 不是写得多么文采飞扬，而是能逼着 AI 把它那点概率计算的能力用到刀刃上，自己给自己当审核员。
