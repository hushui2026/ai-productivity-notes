豆包 MCP：让 AI 直接操作你的常用工具，效率党必备

写方案要查飞书文档，做图表要开 Excel，存文件还要找半天文件夹。每天在不同软件间来回复制粘贴，手都快敲废了。

现在有了**豆包 MCP**，让 AI 直接连通你的常用工具，一句话就能跨平台搞定所有操作。

很多人可能还不知道 MCP 是什么。它的全称是 Model Context Protocol（模型上下文协议）。

简单来说，以前的大模型是个断网的大脑，只能和你对话；现在 **MCP 协议**相当于给这个大脑接上了手脚，让 AI 能直接读写你电脑上的文件、操作你的工作软件。

**豆包 MCP** 就是字节跳动把这个协议落地到了豆包客户端里。你不需要写代码，就能让豆包直接操作你的本地文件、飞书文档等常用工具。

下面教你如何用豆包 MCP 接入常用工具，只需 4 步：

① **更新客户端**
确保你下载了最新版的豆包电脑客户端。旧版本是不支持 MCP 功能的。

② **进入 MCP 市场**
打开豆包客户端，在左侧边栏找到「MCP 工具」或者点击对话框的「+」号，进入工具市场。

[截图：豆包客户端 MCP 工具市场入口]

③ **添加所需工具**
在市场里，你会看到各种工具，比如「本地文件系统」、「飞书文档」、「GitHub」等。找到你日常最常用的，点击「添加」并开启。

比如我经常需要处理本地文档，就添加「本地文件系统」。

④ **授权与配置**
添加工具后，系统会提示你授权。本地文件系统需要指定允许 AI 访问的文件夹路径（比如桌面或某个工作目录），飞书则需要扫码登录授权。

[截图：本地文件系统路径授权配置界面]

完成配置后，你在豆包里发指令，AI 就会自动调用这些工具。

给大家分享一个我常用的自动化办公 Prompt 模板：

```text
你现在是我的工作助理。请帮我完成以下任务：
1. 读取【本地文件系统】中"D:/工作周报/本周数据.xlsx"里的内容；
2. 提取其中新增用户数和转化率两个核心指标；
3. 将提取的数据整理成一段简短的总结；
4. 将这段总结写入【飞书文档】中名为"团队周报汇总"的文档末尾。
```

[截图：豆包根据 Prompt 自动调用工具执行任务的过程]

在这个流程里，豆包会自己打开本地 Excel 读取数据，然后自动切换到飞书写入文档。你只需要喝口水，等它执行完就行。

传统方式下，找文件、看数据、复制粘贴到飞书，来回切换窗口至少需要 5 分钟，还容易看错数字。用豆包 MCP，发一句话指令，30 秒全自动完成，准确率 100%。

**小贴士：**

1. **安全设置**：在配置本地文件系统时，千万不要把 C 盘根目录或包含隐私文件的路径开放给 AI。建议单独建一个"AI 工作区"文件夹，只授权这个路径。
2. **明确工具**：如果你的 Prompt 可能涉及多个工具，最好在指令里指明用哪个工具（如上文模板中的【本地文件系统】），这样 AI 调用会更精准，不会胡乱猜测。
