# Ctrl+S存出乱码文件夹？用Kage把网页打包成1个离线文件，断网也能看

老王昨天在微信上疯狂戳我。

“完了完了，我前天让那个什么大模型帮我生成的竞品分析报告，当时看着可好了，今天想导出来发给老板，网页打不开了！”

他说链接点进去是个404。我一点都不意外。

现在的网页真的很脆弱。尤其是AI生成的长篇内容，很多时候依托于临时的对话链接，或者服务器缓存，过两天就失效了。再比如你看到一个很好的网页资料，过阵子人家网站改版，原来的内容直接没影了。有时候碰上防删帖，人家直接把文章删了，你连个证据都留不下。

“你没Ctrl+S存一份吗？”我问。

老王发来个苦笑的表情。他存了。但打开一看，排版全乱了，图片也是个叉。

他还试过用微信收藏，结果因为网页里有动效，收藏进去直接卡死。

讲真，浏览器自带的保存网页功能，基本是个半残废。存下来的东西要么是一堆散乱的文件夹加一个HTML文件，发给别人还得打包压缩；要么就是丢三落四，稍微有点动态交互的网页，存下来根本没法看。复制粘贴到Word里？那更别提了，表格错位，代码块全崩。

我以前也用过印象笔记或者 Notion 的那个网页剪藏插件。说实话，看情况。有时候剪藏进去排版还可以，但大部分时候，只要网页稍微用了点前端框架，剪藏进去就是一堆乱码，或者图片直接丢了。而且它还得依赖软件本身，你要是换了个不支持插件的浏览器，或者哪天软件服务器挂了，你这存档等于白存。

这时候就得靠 Kage 了。

“Kage是个啥？”老王问。

这是个能做网页离线保存的神器。它能把一个网页，连同它所有的图片、CSS样式、JavaScript交互，甚至AI生成的复杂排版，全部打包成一个独立的HTML文件。

注意，是单文件打包。不管原网页多复杂，最后就给你一个文件。双击就能打开，断网也能看。防删帖和做信息存档的时候，这玩意简直是保命用的。这种直接生成单文件 HTML 的做法，是最底层的，也是最保险的。只要这世界上还有浏览器，你这个文件就能打开。

“听着不错，但装起来是不是很麻烦？我不懂写代码啊。”老王有点发怵。

你不需要懂代码。虽然它是个命令行工具，但就那一两条命令，复制粘贴改个网址就行。

我让他先去装个 Node.js 环境，因为 Kage 是基于这个环境跑的。去 Node 官网，下载那个长期维护版（LTS），一路点下一步装好就行，不用配置啥。

然后在电脑上按 Win+R，输入 cmd 回车，弹出来那个黑框框。在里面输入这一行：

`npm install -g kage`

等它跑完进度条，就算装好了。

老王跟着弄完，回我：“然后呢？怎么把那个AI生成的网页弄下来？”

很简单。你把那个网页的网址复制下来，然后在黑框框里敲命令。

```bash
kage --output [你要保存的文件名].html [网页链接]
```

比如你要存那个AI报告，就是 `kage --output report.html https://xxx.com/chat/123`。回车之后等个几秒钟，它会在你当前电脑的文件夹里生成一个 report.html 文件。

老王试了一下，过了一会儿发来消息：“神了！真就一个文件，双击打开排版全都在，图也全出来了。连那个折叠的代码块都能展开！这个能保存那种很长的公众号文章吗？”

能。而且特别适合存长文。你想想，有些干货文章你今天看了觉得不错，过两天可能因为各种原因被删了。用 Kage 跑一下，哪怕原链接变成“该内容已被发布者删除”，你手里依然捏着一份完整的单文件存档。

“那如果网页需要登录才能看呢？比如我们公司那个内部系统，还有那个什么竞品的后台数据，我想存个档。”老王来了兴致。

这就是 Kage 另一个牛逼的地方了。它可以直接把你当前浏览器里已经登录的状态抓下来。

我跟他说，你平时用的浏览器里，应该都装了类似 Cookie 导出的插件吧？没有的话去装个叫 EditThisCookie 的插件，或者类似的。把你需要登录的那个网页的 Cookie 导出成文本。然后在命令里加个参数：

```bash
kage --output [你要保存的文件名].html --cookie "[你导出的Cookie文本]" [网页链接]
```

这样它就会带着你的登录状态去抓取页面，连那些需要特定权限才能看的东西，也能给你扒下来打包。做竞品分析的时候，把人家官网改版前的样子存一份，以后写报告随时能翻出来对比，不用到处找截图。

老王直呼好家伙，说他以后看长文章再也不用担心看着看着网页挂了。

不过他又问了个问题：“这东西每次都要敲命令吗？我记性不好啊。而且我看那种带很多图的网页，加载挺慢的，它能抓全吗？”

其实你弄熟了就几秒钟的事。你要是真嫌麻烦，可以把那个命令写在一个 txt 记事本里，每次要用的时候，改一下网址和文件名，直接粘贴到命令行里。跟复制粘贴没区别。

至于图片加载慢的问题，Kage 有个等待机制。有些网页它是那种你往下滑才加载图片的，也就是常说的懒加载。Kage 在抓取的时候，会模拟浏览器往下滑的动作，把内容都触发出来。所以哪怕是很长的网页，它也能给你完完整整地扒下来。

还有最重要的一点。就是你跟AI聊天的时候，有些回答结构特别复杂，里面有代码块，有表格，甚至还有它渲染出来的图表。这种内容如果直接复制粘贴，排版一定会崩。你这时候用 Kage 把整个对话页面抓下来，所有的格式都原封不动地保留。这就相当于你自己建了一个私人离线知识库，而且是零成本的那种。发给别人也方便，就一个 HTML 文件，谁都能打开，不用装什么乱七八糟的阅读器。

老王最后把那份报告顺利打包，发给老板了。他说老板平时出差多，在高铁上信号不好，打不开网页链接。给他发个单文件，在本地直接双击就看了，连图都能秒开，体验特别好。

他问我，这种工具是不是很小众。

确实，比起那些花里胡哨的笔记软件，这种命令行工具显得有点极客。但讲真，很多看起来粗糙的工具，恰恰是最管用的。它不跟你搞什么云同步、会员充值，就是单纯地把网页变成文件交到你手里。你要做信息存档，防删帖，或者离线看资料，这招绝对够用。

信息这东西，放在别人服务器上总归是不踏实的。抓在自己手里，才是真的存档。
